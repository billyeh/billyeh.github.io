Notes on problems encountered while deploying a local-first app, and potential solutions

  1. Storing app state locally: for Flutter, Hive and Isar are older options, while Drift is a newer one.
  2. Authentication: for Flutter, you can use Firebase Auth, Supabase Auth, or roll your own.
  3. Backend server for storing state remotely: for Flutter, you can use something like PowerSync, Pocketbase, any IAAS provider, or Supabase.
  4. Deploying the backend: you can use Coolify, which seems a little more batteries-included, or roll your own using a setup like Caddy and DuckDNS.
